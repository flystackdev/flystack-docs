---
title: "Airline Routes"
api: "GET https://api.flystack.dev/v1/routes"
description: "Explore airline routes and planned schedules (non real-time) to build connections and future itineraries."
---

The Routes dataset contains planned airline schedules that are operated by airlines. This is **not** real-time data and does not include live flight status.

Together with the [Flight Schedules API](/api-reference/endpoints/schedules), routes data can help you predict schedules further into the future and build connection logic (days of week, departure times, aircraft type, terminals, etc.).

### Authentication

<ParamField header="x-api-key" type="string" required={true}>
  Your FlyStack API token. You can create one from the dashboard (see [Authentication](/authentication)).
</ParamField>

### Query parameters

#### Route filters

<ParamField query="dep_iata" type="string" required={false}>
  Filter by departure airport IATA code (e.g., `CMB`).
</ParamField>

<ParamField query="dep_icao" type="string" required={false}>
  Filter by departure airport ICAO code (e.g., `VCBI`).
</ParamField>

<ParamField query="arr_iata" type="string" required={false}>
  Filter by arrival airport IATA code (e.g., `AUH`).
</ParamField>

<ParamField query="arr_icao" type="string" required={false}>
  Filter by arrival airport ICAO code (e.g., `OMAA`).
</ParamField>

<ParamField query="airline_iata" type="string" required={false}>
  Filter by airline IATA code (e.g., `UL`).
</ParamField>

<ParamField query="airline_icao" type="string" required={false}>
  Filter by airline ICAO code (e.g., `ALK`).
</ParamField>

<ParamField query="flight_icao" type="string" required={false}>
  Filter by flight ICAO code-number (e.g., `ALK2265`).
</ParamField>

<ParamField query="flight_iata" type="string" required={false}>
  Filter by flight IATA code-number (e.g., `UL2265`).
</ParamField>

<ParamField query="flight_number" type="string" required={false}>
  Filter by flight number only (e.g., `2265`).
</ParamField>

#### Response shaping & pagination

<ParamField query="_fields" type="string" required={false}>
  Comma-separated list of fields to return (e.g., `airline_iata,flight_number`).
</ParamField>

<ParamField query="limit" type="number" required={false}>
  Max number of results per response (limits may depend on your plan).
</ParamField>

<ParamField query="offset" type="number" required={false}>
  Pagination offset (0+). Use with `limit` when a response indicates there are more results.
</ParamField>

<Note>
  For best results, provide at least one filter (airport, airline, or flight code).
</Note>
