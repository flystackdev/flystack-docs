---
title: "Flight Delays"
api: "GET https://api.flystack.dev/v1/delays"
description: "Monitor live flight delays worldwide or filter by airport, airline, or flight."
---

<Note>
  Beta: This endpoint reflects live delays for departing or arriving flights. Coverage may vary.
</Note>

Real-time delay data helps you monitor disruption anywhere in the world, or focus on a specific airport/route. Itâ€™s especially useful for airport transfers, travel insurance, and delay compensation workflows.

### Authentication

<ParamField header="x-api-key" type="string" required={true}>
  Your FlyStack API token. You can create one from the dashboard (see [Authentication](/authentication)).
</ParamField>

### Query parameters

#### Required

<ParamField query="delay" type="number" required={true}>
  Minimum delay (in minutes). Must be greater than 30.
</ParamField>

<ParamField query="type" type="string" required={true}>
  Flight type:
  - `departures`
  - `arrivals`
</ParamField>

#### Optional filters

<ParamField query="dep_iata" type="string" required={false}>
  Filter by departure airport IATA code.
</ParamField>

<ParamField query="dep_icao" type="string" required={false}>
  Filter by departure airport ICAO code.
</ParamField>

<ParamField query="arr_iata" type="string" required={false}>
  Filter by arrival airport IATA code.
</ParamField>

<ParamField query="arr_icao" type="string" required={false}>
  Filter by arrival airport ICAO code.
</ParamField>

<ParamField query="airline_icao" type="string" required={false}>
  Filter by airline ICAO code.
</ParamField>

<ParamField query="airline_iata" type="string" required={false}>
  Filter by airline IATA code.
</ParamField>

<ParamField query="flight_icao" type="string" required={false}>
  Filter by flight ICAO code-number.
</ParamField>

<ParamField query="flight_iata" type="string" required={false}>
  Filter by flight IATA code-number.
</ParamField>

<ParamField query="flight_number" type="string" required={false}>
  Filter by flight number only.
</ParamField>

#### Response shaping & pagination

<ParamField query="_fields" type="string" required={false}>
  Comma-separated list of fields to return (e.g., `airline_iata,flight_number`).
</ParamField>

<ParamField query="limit" type="number" required={false}>
  Max number of results per response (limits may depend on your plan).
</ParamField>

<ParamField query="offset" type="number" required={false}>
  Pagination offset (0+). Use with `limit` when a response indicates there are more results.
</ParamField>

### Response

The endpoint returns a list of delayed flights.
